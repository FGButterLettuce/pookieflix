# Onboarding Wizard Visual Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace PookieFlix's text-heavy, dark-only onboarding wizard with a light/dark-togglable, visual-first wizard that front-loads prerequisites and helps domain-less couples get a domain, built on top of the marketing site's already-validated "Pookie" design system instead of inventing a new one.

**Architecture:** A new `ThemeContext` (React context + `data-theme` attribute on `<html>`) drives light/dark tokens defined in `client/src/index.css`, ported directly from `pookieflix-site`'s Pookie/You mode tokens. `Setup.tsx`'s step machine gains a `tunnelPhase` sub-state that inserts a domain-check → name-collection → suggestion-cards flow before the existing Cloudflare instructions. Domain suggestions are generated by a pure, unit-tested function with no network calls. Two new persisted fields (`USER_NAME`, `PARTNER_NAME`) round-trip through the existing `/api/setup` and `/api/settings` routes.

**Tech Stack:** React 18 + TypeScript (client, Vite), Fastify (server), `node:test` + `tsx` for tests (matching the server's existing pattern — the client gains this same pattern, since it currently has no test runner at all).

## Global Constraints

- Light theme tokens (from `pookieflix-site`'s `[data-mode="pookie"]`): `--bg:#eae6ef` `--bg2:#f5f1f8` `--surface:#ffffff` `--border:rgba(52,32,63,0.1)` `--text:#34203f` `--text-muted:#6b566f` `--text-subtle:#9a7bb5` `--accent:#e0457f` `--accent-hover:#c73569` `--accent-rgb:224,69,127`.
- Dark theme tokens (from `[data-mode="you"]`, with the contrast fix validated in this session's brainstorm): `--bg:#130b1c` `--bg2:#1a1028` `--surface:#1f1029` `--border:rgba(255,245,239,0.08)` `--text:#fff5ef` `--text-muted:rgba(255,245,239,0.68)` (NOT `#9a7bb5` — that was the too-low-contrast original) `--text-subtle:#9a7bb5` `--accent:#ff7fab` `--accent-hover:#ff9dbc` `--accent-rgb:255,127,171`.
- Type scale: step titles 22px/Figtree 900/`letter-spacing:0.01em`/`margin-bottom:16px`; body text 15px/`line-height:1.6`; numbered instructions 15px/`line-height:1.8`.
- Buttons: primary CTAs are pill-shaped (`border-radius:999px`) with `box-shadow:0 4px 16px rgba(var(--accent-rgb),0.3)`.
- Copy voice: lowercase, warm, second-person, dialed back from the marketing site's exclamation-heavy register (this is a wizard to complete, not a pitch to read).
- No registrar API integration, no new server route for domain suggestions — client-side pure function only, no network calls, no API keys.
- Do not fabricate a Spaceship URL prefill parameter — it isn't publicly documented and their search page 403s automated fetches. Use a bare link to `https://www.spaceship.com/domain-search/` plus a separate "copy name" button.
- `USER_NAME`/`PARTNER_NAME` are not secrets — no masking in API responses (unlike `OPENSUBTITLES_API_KEY`).
- Theming is app-wide (Home/Room inherit it too), not scoped to Setup/Settings only — confirmed since `index.css`'s CSS variables are already global.
- Every change gets verified against a real Docker build on `twogether-box` (LAN, `192.168.0.91`, SSH alias `twogether-box`) before this is considered done — this project has no client-side UI test harness, so build success (`tsc && vite build`) plus real-device manual verification is the established substitute for component tests, per the project's own testing conventions (see previous session's KB entry) and Niranjan's standing instruction.
- Hold off on the release pipeline for this work until Niranjan has seen and approved the new onboarding in the running app.

```ts
export type Theme = 'light' | 'dark';

export function resolveInitialTheme(storedValue: string | null, systemPrefersDark: boolean): Theme {
  if (storedValue === 'light' || storedValue === 'dark') return storedValue;
  return systemPrefersDark ? 'dark' : 'light';
}
```

- [ ] **Step 6: Create the ThemeContext**

Create `client/src/theme/ThemeContext.tsx`:

```tsx
import { createContext, useContext, useEffect, useState, type ReactNode } from 'react';
import { resolveInitialTheme, type Theme } from './resolveInitialTheme';

const STORAGE_KEY = 'pookieflix-theme';

interface ThemeContextValue {
  theme: Theme;
  setTheme: (t: Theme) => void;
}

const ThemeContext = createContext<ThemeContextValue | null>(null);

export function ThemeProvider({ children }: { children: ReactNode }) {
  const [theme, setThemeState] = useState<Theme>(() =>
    resolveInitialTheme(
      localStorage.getItem(STORAGE_KEY),
      window.matchMedia('(prefers-color-scheme: dark)').matches
    )
  );

  useEffect(() => {
    document.documentElement.setAttribute('data-theme', theme);
  }, [theme]);

  useEffect(() => {
    if (localStorage.getItem(STORAGE_KEY)) return; // user already made an explicit choice
    const mql = window.matchMedia('(prefers-color-scheme: dark)');
    const onChange = (e: MediaQueryListEvent) => setThemeState(e.matches ? 'dark' : 'light');
    mql.addEventListener('change', onChange);
    return () => mql.removeEventListener('change', onChange);
  }, []);

  const setTheme = (t: Theme) => {
    localStorage.setItem(STORAGE_KEY, t);
    setThemeState(t);
  };

  return <ThemeContext.Provider value={{ theme, setTheme }}>{children}</ThemeContext.Provider>;
}

export function useTheme(): ThemeContextValue {
  const ctx = useContext(ThemeContext);
  if (!ctx) throw new Error('useTheme must be used within a ThemeProvider');
  return ctx;
}
```

```ts
export interface DomainSuggestion {
  domain: string;
  featured?: boolean;
}

function slug(name: string): string {
  return name.trim().toLowerCase().replace(/[^a-z0-9]/g, '');
}

export function generateDomainSuggestions(userName: string, partnerName: string): DomainSuggestion[] {
  const you = slug(userName) || 'you';
  const pookie = slug(partnerName) || 'pookie';
  const blend = `${you}${pookie}`;

  const candidates: DomainSuggestion[] = [
    { domain: `${blend}.com` },
    { domain: `our.movienight.app`, featured: true },
    { domain: `${blend}watch.xyz` },
    { domain: `watch.with${pookie}.com` },
    { domain: `${blend}.app` },
    { domain: `${you}and${pookie}.com` },
    { domain: `our.together.app` },
    { domain: `${pookie}and${you}watch.xyz` },
  ];

  const seen = new Set<string>();
  return candidates.filter(s => {
    if (seen.has(s.domain)) return false;
    seen.add(s.domain);
    return true;
  });
}
```
